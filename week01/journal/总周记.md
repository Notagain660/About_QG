  - 本地仓库：你电脑上的缓存（默认 `~/.m2/repository`）

##### ->api实现-> 添加工具类
- `@RestController` 告诉 Spring 这个类是一个控制器，里面的每个方法都会处理 HTTP 请求，并且返回的数据（比如 `MapperResult`、`User`）会直接写成 JSON 响应给前端，而不是跳转到某个页面。  
- `@RequestMapping("/api")` 给这个控制器的所有方法加了一个公共的 URL 前缀 `/api`，这样在方法上只要写 `/login`，实际访问就是 `/api/login`，方便统一管理接口路径。
- 请求方法：get post put delete
- `@RequestParam` 用于从 URL 的查询字符串（如 `?id=123`）或表单数据中获取单个参数值，常用于 GET 请求或简单的表单提交。（按状态查看报修单用到；loginrepairorder可以被路径变量取代了）  
- `@RequestBody` 用于读取 HTTP 请求体（body）中的 JSON/XML 等数据，并将其自动转换为 Java 对象，常用于 POST、PUT 请求中接收复杂数据（如前端传来的整个用户对象）。
- @PathVariable从 URL 路径中获取变量值（对单个表单操作用到，包括删除和评论）
- 使用泛型灵活控制传进线程的参数
- 写的过程发现返回code、data和message不好控制，封装成为logindata来控制每次的返回信息和data类型；异常处理使用日志打印，思路仿照控制台项目的异常处理，但是不单处理runtimeexception
