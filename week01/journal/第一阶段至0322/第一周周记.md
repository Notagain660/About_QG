### 0327
- 原控制台程序中把所有能更新的数据放在一起的update在controller里面需要分开成更新宿舍和修改密码（update逻辑不变，controller更新宿舍时密码get，更新密码时宿舍get）
- `<T>`泛型可以根据需要传不同的数据类型（如list，引用类型，基本类型/object）
- get不用于requestbody，并且请求参数不是json，**`Map<Long, Object>`** 作为参数，Spring 无法自动将 JSON 键（字符串）映射为 `Long`
```
|查询当前登录用户信息|无需参数|`GET /profile`|
|根据 ID 查询资源|路径变量|`GET /user/{id}`|
|根据条件筛选（多参数）|请求参数|`GET /orders?status=pending&page=1`|
|创建/更新资源（传复杂数据）|请求体|`POST /user` 或 `PUT /user`|
```

- 原先的loginrepairorder可以被路径变量取代了，不需要用户输入但是操作是post的也是
- 为了防止恶意攻击后端依旧要校验（如取消报修单功能，还有查询功能时的身份校验）
- @requestparam可以实现/api/user/repairorders?status=pending类型的请求，表示从 URL 的查询字符串中获取参数，例如 `?status=pending`。
    - 可以在一个方法里根据不同的status查询不同状态的报修单（管理员）
    - `required = false` 表示这个参数不是必须的。如果请求中没有 `status` 参数，`status` 变量就会是 `null`（而不是报错）
    - 不是必填项才能加required=false
- 加了几个用户是不是本人/是不是学生或管理员身份的校验
